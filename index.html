Setup Instructions

Getting Started

Follow these steps to set up ARXplore on your system and start analyzing ARXML files with AI-powered insights.

Prerequisites

  • Python 3.8 or higher
  • Gemini API key
  • Git
  • Internet connection for AI features
  • 4GB RAM minimum (8GB recommended)

Installation Steps

Step 1: Clone the repository
Terminal 
git clone https://github.com/yourusername/arxplore.git
cd arxplore
Step 2: Create a .env file

Create a file named .env in the project root directory and add your Gemini API key:

.env 
GOOGLE_API_KEY=your_gemini_api_key_here
How to get a Gemini API key:
  1. Visit Google AI Studio
  2. Sign in with your Google account
  3. Click on "Create API Key"
  4. Copy the generated key to your .env file
Step 3: Create a virtual environment
Command Prompt 
python -m venv myenv
myenv\Scripts\activate
Terminal 
python3 -m venv venv
source venv/bin/activate
Step 4: Install requirements
Terminal 
pip install -r requirements.txt
Step 5: Running the Application

Simply double-click the run_app.bat file or run it from the command line:

Command Prompt 
run_app.bat

This will start both the Streamlit app and the Compare app in separate minimized windows. You can find them in your taskbar.

Execute the run script:

Terminal 
chmod +x run_app.sh
./run_app.sh

This will start both the Streamlit app and the Compare app as background processes. The script will display their process IDs and instructions on how to stop them when you're done.

You can run each application separately:

Terminal 
# For the main Streamlit application
streamlit run streamlit.py

# For the Compare application
python compare/app.py

The main Streamlit application will be available at http://localhost:8501 in your web browser. The Compare application will be available at http://localhost:7465.

Development Approaches

Approach 1

This approach focuses on providing a web interface for ARXML file visualization and comparison with interactive tree views.


1. Project Motivation

ARXplore was developed with the objective of enabling efficient comparison of AUTOSAR ARXML (AUTOSAR XML) files at the node level, and presenting the differences through a user-friendly, interactive interface. When two ARXML/XML files are uploaded, the tool identifies structural and content-level changes?such as additions, deletions, or modifications?and displays them side by side in a hierarchical format. This helps users quickly understand how the files differ without manually inspecting the raw XML.

Primary Objectives:
  • To enable intuitive exploration of ARXML files via a hierarchical, tree-based interface.
  • To perform structural comparison between two ARXML/XML files and highlight changes.
  • To generate AI-powered summaries of file differences in natural language.

4. Development Phases

Phase A Input Handling and XML Parsing
  • Implemented file upload functionality using Streamlit's native uploader.
  • Parsed ARXML/XML files using Python's ElementTree and converted them into nested dictionary structures.
  • These structures were serialized into JSON-compatible formats for use in the frontend.
Phase B Hierarchical Tree Rendering
  • Built a recursive tree rendering function that maps XML nodes to a collapsible tree view.
  • Maintained parent-child relationships and preserved attribute-level information for each node.
  • Integrated expand/collapse toggling to manage large file structures efficiently.
Phase C Differencing Engine

Developed a recursive tree comparison algorithm:

  • Compares two parsed XML trees node-by-node.
  • Identifies insertions, deletions, and value modifications.
  • Flags structural differences and records them in a diff object.

Implemented key heuristics to handle optional or reordered elements commonly found in AUTOSAR schemas.

Phase D Difference Visualization

Visualized file differences using color-coded indicators:

  • Green: Nodes present only in the second file (additions).
  • Red: Nodes present only in the first file (deletions).
  • Yellow: Nodes modified between versions.

Supported side-by-side synchronized tree views for parallel comparison.

Enabled synchronized scrolling and highlighting across the two views.

Phase E Natural Language Summary via AI
  • Constructed structured prompts from the diff object.
  • Passed prompts to a connected LLM (e.g., Gemini or GPT) to generate natural language descriptions of the differences.
  • Integrated the AI-generated output within the UI for user review.
Phase F UI Integration and Workflow Management

Used Streamlit to orchestrate user interactions, application state, and dynamic content rendering.

Managed session-level context to support features such as:

  • Diff mode toggling.
  • View synchronization.
  • AI summary on demand.

5. Deliverable Summary (v1.0)

ARXplore v1.0 delivers the following functionality:

  • Support for single and dual ARXML/XML file uploads.
  • Visual tree-based representation of file contents.
  • Structural comparison and visualization of differences.
  • AI-generated natural language summaries of file changes.
  • Lightweight, browser-accessible UI with minimal setup.

6. Conclusion

ARXplore represents a hybrid solution combining traditional XML processing techniques with AI-driven insights and interactive visualization. Its modular architecture enables both technical accuracy and user accessibility, making it a valuable tool for AUTOSAR-based development environments.

3. Development Phases and Features

Phase A Core Functionalities

The tool was developed with the primary objective of offering advanced validation and comparison capabilities. The key features of ARXplore v1.0 include:

Feature File Analysis
  • Element Statistics: Provides statistical insights into the distribution of ARXML elements.
  • Attribute Analysis: Identifies and presents the most common attributes across files.
  • Validation Results: Detailed error reporting aligned with AUTOSAR standards, ensuring compliance with relevant specifications.
Feature Schema Validation
  • Version Detection: Automatically identifies the schema version of uploaded ARXML files.
  • XSD Validation: Performs full validation against official AUTOSAR XSD schemas.
  • Version Comparison: Highlights mismatches between the uploaded file and the selected AUTOSAR schema version.
Feature AI Analysis
  • Smart Comparison: Leverages AI for semantic difference analysis, enhancing the accuracy of file comparisons.
  • Data Extraction: Uses natural language queries to extract specific data, such as listing all ECU instances.
  • Comprehensive Review: Provides AI-driven architectural and safety assessments based on file configurations.
Feature AI Assistant
  • Context-Aware Chat: Enables users to ask specific questions about their files or AUTOSAR concepts, receiving AI-powered responses.
  • Technical Support: Provides detailed explanations of validation errors, helping users address potential issues in their ARXML files.
Feature Insights Dashboard
  • Visual Analytics: Displays interactive charts showcasing element distributions within ARXML files.
  • Trend Analysis: Identifies common patterns across configurations, assisting users in understanding recurring design choices.
Phase B Key Components

The ARXplore tool was designed with modular architecture, incorporating several key components for efficient processing and analysis of ARXML files:

Component ARXMLProcessor Class
  • Handles XML parsing, namespace management, and node extraction.
  • Tracks paths and compares nodes to detect changes between files.
  • Implements validation rules and comparison algorithms for structural analysis.
Component Validation Framework
  • Schema Validation: Ensures the structure of ARXML files conforms to the selected XSD schema version.
  • Rule-Based Validation: Implements custom AUTOSAR-specific validation rules, such as checking the validity of mandatory attributes in ARXML elements.
  • Custom Validation Functions: Includes additional checks, such as UUID format and attribute length constraints.
Component AI Integration
  • Integrates Google's Gemini AI for enhanced file analysis. The AI leverages context-specific prompts and provides semantic analysis, including functional impact, safety implications, performance considerations, and recommended actions.
  • Response Handling: Ensures AI-generated responses are presented in a structured, user-friendly manner (e.g., using markdown formatting).
Visualization System
  • Plotly is used to generate dynamic charts and graphs for visual data representation.
  • Custom Streamlit components are employed to create interactive and responsive user interfaces, incorporating custom CSS and animated elements for an enhanced user experience.

4. AI Integration and Prompt Engineering

The AI system within ARXplore follows a multi-stage approach for analysis and response generation:

Stage 1 Context Preparation

Prepares necessary context, including file statistics, sample nodes, validation errors, and comparison matrices.

Stage 2 Prompt Engineering

Stage 3 Response Handling

AI responses are delivered in real-time, displayed via a streaming interface with markdown formatting to ensure clarity and coherence.

5. Validation Framework

ARXplore uses a multi-level validation framework, which includes:

Level 1 Schema Validation

Full validation against the XSD schema ensures that ARXML files adhere to AUTOSAR structural guidelines.

Level 2 Rule Validation

Custom AUTOSAR-specific rules are applied to validate attributes like UUID, SHORT-NAME, and other key elements.

Level 3 AI Validation

Beyond structural and rule validation, AI validation performs semantic checks, evaluating the functional, safety, and performance impacts of configuration changes.
